## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- **No orphan code**: every planned endpoint, entity, field, service,
  repository, integration, frontend/auth change, and behavior is traced to a
  spec requirement, business rule, user story, or acceptance criterion.
- **Layered architecture**: source layout separates routes, controllers,
  services, repositories, schemas, errors, and tests. Controllers delegate to
  services; services use repositories or equivalent ports.
- **Validation and failures**: every external input has planned Zod validation;
  business failures define stable error codes and the API error response uses
  `{ "error": { "code": "...", "message": "..." } }`.
- **Testing discipline**: important business rules have Vitest coverage planned;
  infrastructure is isolated with mocks, fakes, or in-memory adapters unless an
  integration test is explicitly justified.
- **Scope exclusions**: frontend and authentication are excluded unless this
  feature specification explicitly includes them.
